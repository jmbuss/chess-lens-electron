# Database Model Creation Guide

This guide explains how to create database models in the Chess Lens application, using the `User` model as a reference example.

## Overview

Database models in this project follow a consistent pattern:
- They implement the `BaseModel` interface
- They use SQLite via `better-sqlite3`
- They provide static methods for CRUD operations
- They handle table initialization and migrations
- They use TypeScript for type safety

## File Structure

Models are typically organized in the `src/database/` directory. The User model follows this structure:

```
src/database/
  user/
    model.ts      # The model class with CRUD operations
    types.ts      # TypeScript interfaces for the model data
  models/
    BaseModel.ts  # Base interface that all models implement
```

## Step-by-Step Guide

### 1. Create Type Definitions

First, define the TypeScript interface for your model data in a `types.ts` file:

```typescript
// src/database/user/types.ts
export interface UserData {
  id?: number
  firstName: string
  lastName: string
  email: string
  createdAt?: string
  updatedAt?: string
}
```

**Key points:**
- `id` is optional (auto-generated by SQLite)
- `createdAt` and `updatedAt` are optional (set by database defaults)
- All other fields should be required unless they're truly optional

### 2. Create the Model Class

Create a model class that implements `BaseModel`:

```typescript
// src/database/user/model.ts
import type Database from 'better-sqlite3'
import type { BaseModel } from '../models/BaseModel'
import type { UserData } from './types'

export class UserModel implements BaseModel {
  // Implementation below...
}
```

### 3. Implement `initializeTables`

This method creates the table and handles migrations:

```typescript
initializeTables(db: Database.Database): void {
  // Create table if it doesn't exist
  db.exec(`
    CREATE TABLE IF NOT EXISTS users (
      id INTEGER PRIMARY KEY AUTOINCREMENT,
      firstName TEXT NOT NULL,
      lastName TEXT NOT NULL,
      email TEXT NOT NULL UNIQUE,
      createdAt TEXT NOT NULL DEFAULT (datetime('now')),
      updatedAt TEXT NOT NULL DEFAULT (datetime('now'))
    )
  `)

  // Migrate existing table: check for missing columns and add them
  const tableInfo = db.pragma('table_info(users)') as Array<{ name: string }>
  const existingColumns = new Set(tableInfo.map((col) => col.name))

  // Add columns if missing (with defaults for existing rows)
  if (!existingColumns.has('firstName')) {
    db.exec('ALTER TABLE users ADD COLUMN firstName TEXT NOT NULL DEFAULT ""')
  }
  // ... repeat for other columns

  // Create indexes for faster lookups
  db.exec(`
    CREATE INDEX IF NOT EXISTS idx_users_email ON users(email)
  `)
}
```

**Key points:**
- Use `CREATE TABLE IF NOT EXISTS` to avoid errors on re-initialization
- Always include `id INTEGER PRIMARY KEY AUTOINCREMENT`
- Use `TEXT NOT NULL DEFAULT (datetime('now'))` for timestamp fields
- Add migration logic to handle schema changes
- Create indexes on frequently queried columns (like email, foreign keys)

### 4. Implement Static CRUD Methods

All CRUD operations are static methods that take the database instance as the first parameter.

#### Create

```typescript
static create(
  db: Database.Database,
  userData: Omit<UserData, 'id' | 'createdAt' | 'updatedAt'>
): UserData {
  const stmt = db.prepare(`
    INSERT INTO users (firstName, lastName, email)
    VALUES (?, ?, ?)
  `)

  const result = stmt.run(userData.firstName, userData.lastName, userData.email)
  return this.findById(db, result.lastInsertRowid as number)!
}
```

**Key points:**
- Use `Omit` to exclude auto-generated fields from the input type
- Use prepared statements with `?` placeholders (prevents SQL injection)
- Return the created record by fetching it with `findById`

#### Find by ID

```typescript
static findById(db: Database.Database, id: number): UserData | null {
  const stmt = db.prepare('SELECT * FROM users WHERE id = ?')
  return stmt.get(id) as UserData | null
}
```

#### Find by Unique Field

```typescript
static findByEmail(db: Database.Database, email: string): UserData | null {
  const stmt = db.prepare('SELECT * FROM users WHERE email = ?')
  return stmt.get(email) as UserData | null
}
```

#### Update

```typescript
static update(
  db: Database.Database,
  id: number,
  updates: Partial<Omit<UserData, 'id' | 'createdAt'>>
): UserData | null {
  const fields: string[] = []
  const values: any[] = []

  if (updates.firstName !== undefined) {
    fields.push('firstName = ?')
    values.push(updates.firstName)
  }
  if (updates.lastName !== undefined) {
    fields.push('lastName = ?')
    values.push(updates.lastName)
  }
  if (updates.email !== undefined) {
    fields.push('email = ?')
    values.push(updates.email)
  }

  if (fields.length === 0) {
    return this.findById(db, id)
  }

  fields.push("updatedAt = datetime('now')")
  values.push(id)

  const stmt = db.prepare(`
    UPDATE users 
    SET ${fields.join(', ')}
    WHERE id = ?
  `)

  stmt.run(...values)
  return this.findById(db, id)
}
```

**Key points:**
- Use `Partial` to allow updating only specific fields
- Exclude `id` and `createdAt` from updates
- Always update `updatedAt` timestamp
- Build the SET clause dynamically based on provided fields
- Return the updated record

#### Delete

```typescript
static delete(db: Database.Database, id: number): boolean {
  const stmt = db.prepare('DELETE FROM users WHERE id = ?')
  const result = stmt.run(id)
  return result.changes > 0
}
```

**Key points:**
- Return boolean indicating success (changes > 0)

#### Find All

```typescript
static findAll(db: Database.Database): UserData[] {
  const stmt = db.prepare('SELECT * FROM users ORDER BY createdAt DESC')
  return stmt.all() as UserData[]
}
```

**Key points:**
- Use `stmt.all()` for multiple results
- Use `stmt.get()` for single results
- Always cast results to your type

### 5. Register the Model

Register your model in the database initialization (typically in `src/main/main.ts` or wherever `DatabaseService` is instantiated):

```typescript
import { UserModel } from './database/user/model'
import { PlatformAccountModel } from './database/models/PlatformAccount'

const dbService = new DatabaseService([
  new UserModel(),
  new PlatformAccountModel(),
  // Add your new model here
])
```

## Best Practices

1. **Always use prepared statements** - Prevents SQL injection and improves performance
2. **Use TypeScript types** - Define interfaces for all data structures
3. **Handle migrations gracefully** - Check for existing columns before adding new ones
4. **Create indexes** - Add indexes on foreign keys and frequently queried columns
5. **Use foreign keys** - When referencing other tables, use `FOREIGN KEY` constraints
6. **Update timestamps** - Always update `updatedAt` when modifying records
7. **Return typed data** - Cast query results to your TypeScript types
8. **Use UNIQUE constraints** - For fields that should be unique (like email)
9. **Handle nulls properly** - Methods that might not find records should return `null`
10. **Use static methods** - All CRUD operations should be static for consistency

## Foreign Key Relationships

When creating models with foreign keys (like `PlatformAccount` referencing `User`):

```typescript
CREATE TABLE IF NOT EXISTS platform_accounts (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  userId INTEGER NOT NULL,
  platform TEXT NOT NULL,
  platformUsername TEXT NOT NULL,
  createdAt TEXT NOT NULL DEFAULT (datetime('now')),
  updatedAt TEXT NOT NULL DEFAULT (datetime('now')),
  FOREIGN KEY (userId) REFERENCES users(id) ON DELETE CASCADE,
  UNIQUE(userId, platform)
)
```

**Key points:**
- Use `ON DELETE CASCADE` to automatically delete related records
- Use `UNIQUE` constraints for composite unique keys
- Create indexes on foreign key columns

## Testing Your Model

After creating a model:
1. Register it in the `DatabaseService` constructor
2. Restart the application to run `initializeTables`
3. Test CRUD operations to ensure everything works
4. Verify migrations work by checking the database schema
